# Ordo Privacy Policy

Effective date: June 12, 2026
Last updated: June 12, 2026

This Privacy Policy explains how Ordo ("Ordo", "the bot", "we", "us", or "our") accesses, uses, stores, and protects data while providing Discord bot features. Ordo only uses Discord data and user-provided content as needed to operate the bot, maintain safety and moderation features, provide support, and improve reliability.

This policy is not a replacement for Discord's own Terms of Service, Privacy Policy, Community Guidelines, Developer Terms, or Developer Policy. If there is a conflict between this policy and Discord's rules, Discord's rules control.

## Data Ordo May Access

Ordo may access Discord data that is available to the bot through Discord's API and the permissions granted by a server owner, administrator, or user. This may include:

- Server, channel, role, message, user, member, webhook, invite, emoji, sticker, thread, and voice-state IDs.
- Server names, channel names, role names, usernames, display names, avatars, banners, server icons, and public Discord profile information.
- Message content, embeds, attachments, reactions, and message links when needed for enabled features such as message logging, moderation, sticky messages, embed/component builders, docs, templates, support, or audio transcription.
- Member join/leave events, role updates, profile updates, moderation events, audit-log events, invite events, voice events, and command usage.
- User-provided text or files submitted through commands, modals, components, uploads, or support workflows.

Ordo does not request Discord presence intent for online status or activity tracking.

## Data Ordo May Store

Depending on which features are used or enabled in a server, Ordo may store:

- Guild settings, including language, timezone, embed color, configured log channels, ignored log targets, welcome settings, join-role settings, mute settings, asset-channel settings, and branding settings.
- Discord IDs for guilds, users, members, channels, roles, messages, threads, webhooks, invites, suggestions, records, reminders, giveaways, sticky messages, saved embeds, docs, component messages, and related feature objects.
- Moderation records, mute records, staff notes, reasons, evidence references, log message IDs, active lockdown state, and moderation configuration.
- Invite tracking records, including invite codes, inviter IDs, join and leave counts, and member-to-invite mappings.
- User profile settings that users choose to save, such as timezone, about-me text, profile banner URL, preferred embed color, and languages.
- Reminder text, reminder times, repeat settings, and related guild/user/channel IDs.
- Suggestions, suggestion actions, giveaway data, sticky message content, saved docs, saved embed-builder JSON, component-builder configuration, message template text, role notification settings, and related display content.
- Support diagnostics, error reference codes, scrubbed traceback attachments, support command output, and operational status information.
- Discord-hosted asset URLs or Discord message references for assets copied to user DMs or a configured server asset channel.

Ordo does not permanently store uploaded asset files on the bot host except for brief temporary processing. Temporary files are deleted after processing where practical. Some features may copy files into Discord DMs or a configured server asset channel so the asset remains available from Discord-hosted storage.

## How Ordo Uses Data

Ordo uses data to:

- Provide server configuration, moderation, logging, welcome, mute, invite tracking, suggestions, giveaway, reminder, document, embed, component, profile, voice, and utility features.
- Respond to commands and interactions.
- Maintain bot-owned messages, buttons, select menus, persistent views, and builder sessions.
- Produce moderation records, audit-style logs, diagnostics, and support information.
- Troubleshoot errors, prevent abuse, and keep the bot reliable.
- Improve Ordo only through practical debugging, support review, and aggregated or de-identified understanding of issues.

Ordo does not sell, license, or share Discord API data with data brokers, advertising networks, or monetization services.

## Third-Party Services

Ordo primarily operates through Discord and stores most user-facing files or assets on Discord when those files are posted, copied, or attached in Discord.

Ordo may also interact with:

- Gitea or GitHub for owner-only development activity tools, if configured.
- Hosting, storage, backup, or network services controlled by the bot owner for operating Ordo.
- Discord webhooks for support/status messages.

Third-party services are used only as needed to operate, maintain, back up, or support Ordo. Ordo does not intentionally send private Discord data to third parties for advertising or profiling.

## Retention and Deletion

Ordo keeps data only as long as needed for the feature that stored it, for server configuration, for moderation/safety records, for support, or for operational reliability.

Users can delete their own profile data through Ordo's profile privacy/reset controls. Server owners or authorized administrators may request deletion of server-owned Ordo data by contacting support. Data may also be deleted when Ordo leaves a server, when a feature is removed, when Discord or applicable law requires deletion, or when the data is no longer needed for Ordo's stated functionality.

Some data may remain in Discord itself after Ordo removes database references, such as messages, logs, attachments, or asset copies that exist in Discord channels or DMs. Server administrators and users remain responsible for managing Discord messages and files in spaces they control.

To request data access, correction, or deletion, contact Ordo support through the support server: https://discord.gg/rPrg5rur3b

## Security

We use commercially reasonable safeguards to protect Ordo data from unauthorized access, disclosure, loss, or misuse. These safeguards include limiting access to bot credentials and database files, keeping secrets outside tracked source files, using private access-controlled storage, and encrypting backups before they are stored remotely.

The live SQLite database should be stored on an access-controlled host and, where available, on encrypted local storage. Raw database files should not be committed to source control or uploaded to remote services unless the storage is private and encrypted.

No method of electronic storage or transmission is completely secure, so absolute security cannot be guaranteed.

## Children and Minimum Age

Ordo is intended for Discord users who are old enough to use Discord in their country. Ordo is not directed to children under 13 or any higher minimum age required by local law.

## Policy Changes

We may update this Privacy Policy as Ordo changes, as Discord requirements change, or as legal/security needs change. We will take reasonable steps to announce material changes through the Ordo support server or other visible Ordo-controlled channels. Continued use of Ordo after a policy update means continued acceptance of the updated policy.
